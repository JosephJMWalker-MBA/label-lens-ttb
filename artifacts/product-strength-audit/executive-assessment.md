# Executive assessment

## Verdict

Label Lens should narrow substantially. The credible product is the seller-to-internal-review evidence-package path, not a general label maker and not OCR itself. That path has unusually careful evidence boundaries, but it does not yet deliver the human outcome it implies: the agent cannot claim a case, record a decision, request a change, or receive a seller resubmission. Current source therefore supports a durable handoff, not a complete review workflow. Evidence: [`src/app/review/page.tsx`](../../src/app/review/page.tsx), [`src/app/agent/submissions/[id]/page.tsx`](../../src/app/agent/submissions/%5Bid%5D/page.tsx), the GET-only [`src/app/api/agent`](../../src/app/api/agent) routes, and [`src/app/api/package/submit/finalize/route.ts`](../../src/app/api/package/submit/finalize/route.ts), which creates revision 1 and rejects a second submission for the same submission ID.

The current usefulness claim is also unproven. Against a credible manual alternative—a shared folder plus a two-row brand/alcohol evidence worksheet—Label Lens adds provenance, separation, structured handoff, and access controls. It also asks the seller to locate and transcribe both supported fields, mark regions, wait for OCR, resolve discrepancies, and navigate multiple product surfaces. [Issue #38](https://github.com/JosephJMWalker-MBA/label-lens-ttb/issues/38) defines the missing operator-friction comparison. [Issue #57](https://github.com/JosephJMWalker-MBA/label-lens-ttb/issues/57) retains a relevant usefulness-gate intent, but its 61%/57% aggregate is superseded by the current governed alcohol baseline.

The live demo does not currently restore confidence. The bundled legacy fixture completed, but selected `CELLARS` for a declared `M CELLARS` brand and required human review ([LIVE-06](limitations.md#live-observation-log)). One deployed waiting submission observed during the audit appeared to the shared seller and agent accounts, yet its agent detail returned “stored integrity check did not pass” ([LIVE-09](limitations.md#live-observation-log)). This dated observation requires P0 diagnosis, but it does not establish a general integrity-design or persistence failure and its cause is unknown.

## Five strongest assets

1. **Truth and authority are separated.** OCR produces observations, deterministic rules produce outcomes, `not_run` is preserved, and human authority is explicit. Evidence: [`src/server/precheck-service.ts`](../../src/server/precheck-service.ts), [`src/features/package-preparation/package-model.ts`](../../src/features/package-preparation/package-model.ts), [`docs/compliance-readiness-boundary.md`](../../docs/compliance-readiness-boundary.md), and [LIVE-06](limitations.md#live-observation-log).
2. **Seller evidence and machine observations remain distinct.** The package model, exports, and agent detail avoid overwriting seller claims with machine output. Evidence: [`package-model.ts`](../../src/features/package-preparation/package-model.ts), [`package-workflow.ts`](../../src/features/package-preparation/package-workflow.ts), and [`src/server/submissions/detail.ts`](../../src/server/submissions/detail.ts).
3. **The submission foundation is more rigorous than the product loop.** Authenticated ownership, server-recomputed checksums, HMAC integrity, immutable revision snapshots, status events, and idempotent finalization are implemented and tested. Evidence: [`src/app/api/package/submit/finalize/route.ts`](../../src/app/api/package/submit/finalize/route.ts), [`src/lib/integrity.ts`](../../src/lib/integrity.ts), [`src/db/schema.ts`](../../src/db/schema.ts), and [`route.test.ts`](../../src/app/api/package/submit/finalize/route.test.ts).
4. **Research evidence is unusually candid.** The governed corpus reports weak recall, failure classes, latency, false certainty, killed experiments, and the cost of extra OCR passes rather than hiding misses. Evidence: current [`brand metrics`](../brand-evidence-path-diagnosis/metrics.md), current [`alcohol baseline`](../alcohol-digit-ocr-diagnosis/summary.md), and the distinct hard-case [`ocr-region-isolation-benchmark/report.md`](../../docs/ocr-region-isolation-benchmark/report.md).
5. **The bounded package UI is accessible and state-aware.** It requires explicit panel absence, prevents stale analysis from masquerading as current, and keeps only the active stage expanded. Evidence: [`src/app/review/page.tsx`](../../src/app/review/page.tsx), [`package-workflow.ts`](../../src/features/package-preparation/package-workflow.ts), [`tests/e2e/package-preparation.spec.ts`](../../tests/e2e/package-preparation.spec.ts), and [LIVE-03](limitations.md#live-observation-log).

## Five largest confidence-destroying gaps

1. **The workflow stops at read-only agent inspection.** There is no agent decision, change-request, or revision-2 resubmission path. The vocabulary and schema imply future states that no current mutation route can produce. Evidence: [`src/lib/product-language.ts`](../../src/lib/product-language.ts), [`src/db/schema.ts`](../../src/db/schema.ts), [`src/app/api/agent`](../../src/app/api/agent), and [`tests/e2e/auth-portal.spec.ts`](../../tests/e2e/auth-portal.spec.ts).
2. **Usefulness versus manual work is unknown.** No current result shows lower total seller-plus-reviewer time, fewer corrections, or higher trusted throughput. Evidence: [#38](https://github.com/JosephJMWalker-MBA/label-lens-ttb/issues/38), the still-relevant gate intent in [#57](https://github.com/JosephJMWalker-MBA/label-lens-ttb/issues/57), and [`docs/operator-trust-and-throughput.md`](../../docs/operator-trust-and-throughput.md). The #57 aggregate figures are historical, not current.
3. **Machine reading is too weak to be the product promise.** The current governed baselines are 29/105 normalized brand match (27.6%), 33/105 brand top-3 (31.4%), 70/103 alcohol detection (68.0%), and 68/103 parsed alcohol accuracy (66.0%). Alcohol false certainty is 0 and absent-alcohol false positives are 0/13, but 33 present-alcohol cases are still undetected and 35 are not parsed correctly. A distinct five-case vertical mandatory-strip challenge slice remains 0/5 for detection and parsing. Evidence: [`brand metrics`](../brand-evidence-path-diagnosis/metrics.md), [`alcohol baseline`](../alcohol-digit-ocr-diagnosis/summary.md), [`truth correction`](../alcohol-truth-correction/metric-diff.md), and the explicitly separate challenge slice in [`extractor-report.md`](../../docs/extraction-full-corpus/extractor-report.md).
4. **The product presents multiple peers instead of one primary journey.** `/create`, `/review`, `/review/legacy`, and `/learn` all occupy the main navigation; the home page gives unavailable future intents equal product-level prominence. The flows do not share a case or handoff contract. Evidence: [`src/features/home/IntentHub.tsx`](../../src/features/home/IntentHub.tsx), [`src/app/create/page.tsx`](../../src/app/create/page.tsx), [`src/app/review/page.tsx`](../../src/app/review/page.tsx), and [`src/app/review/legacy/page.tsx`](../../src/app/review/legacy/page.tsx).
5. **The public demo, operations, and documentation do not support a durable trust claim.** Shared credentials can create persistent, non-demo-classified artwork records; no reset/deletion/retention job exists; one deployed waiting record failed integrity verification for an unknown reason; core architecture documents still describe a stateless one-image system. Evidence: [`README.md`](../../README.md), [`src/server/submissions/queries.ts`](../../src/server/submissions/queries.ts), [`src/lib/panel-storage.ts`](../../src/lib/panel-storage.ts), [#17](https://github.com/JosephJMWalker-MBA/label-lens-ttb/issues/17), [`docs/architecture.md`](../../docs/architecture.md), and [LIVE-09](limitations.md#live-observation-log).

## Executive decision

Keep and complete the evidence-package loop. Hide the general label scaffold and legacy pre-check from the default product path without deleting them yet. Defer field expansion, broader beverage support, official integration, and broad OCR work until the end-to-end workflow works and Issue #38 demonstrates an advantage over manual review. The approved order is: preserve this corrected audit → diagnose the one live integrity failure → add agent decision/change request → add seller feedback/revision 2 → narrow navigation → repair architecture documentation → instrument Issue #38 → run the paired manual baseline → decide OCR’s role → harden lifecycle and operations before expansion.
