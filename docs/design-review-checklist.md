# Design Review Checklist

Every feature and pull request should be evaluated before merge.

## Problem
- [ ] Solves a stakeholder requirement
- [ ] Fits the defined system boundary
- [ ] Reduces user effort

## Architecture
- [ ] Preserves separation of concerns
- [ ] Adds no unnecessary coupling
- [ ] Uses existing interfaces where appropriate

## Security & Compliance
- [ ] No secrets exposed
- [ ] Data retention policy respected
- [ ] Supports
